# TROPEK API Layer — CRUD Phase Design

**Date:** 2026-03-14
**Status:** Approved
**Scope:** CRUD endpoints for all new domain entities + updated evaluation read endpoints + trend endpoint.
**Deferred:** Evaluation trigger (`POST /evaluations`), batch status, worker refactor — separate brainstorm.
**Depends on:** `docs/superpowers/specs/2026-03-14-tropek-domain-redesign.md` (domain model)
**Supersedes:** Nothing — first API layer spec.

---

## 1. Background

The DB layer redesign (migration 003) added all new ORM models and repositories. The API layer is
currently empty — `main.py` has only a `/health` endpoint. This spec defines the full CRUD API
surface to wire the DB layer to HTTP.

Two bugs in the DB implementation are fixed as part of this work:
- `Asset.tags` should be `Asset.labels` (spec says `labels`, UI expects `labels`)
- `Asset` is missing a `type_name` column (spec defines `type` field)

---

## 2. Design Principles

- **Domain names everywhere** — API response fields match DB column names exactly.
  No mapping aliases (e.g. `period_start` not `start`, `/slo-definitions` not `/slos`).
- **Asset-hub module grouping** — assets, asset groups, and bindings live in one `modules/assets/`
  module. DataSource, SLI, SLO each keep their own module. Evaluations + trend in `quality_gate/`.
- **`PagedResponse[T]` for all list endpoints** — `{"items": [...], "total": N}`.
- **Trend scoped by domain** — `GET /trend?asset_name=&slo_name=&metric=` replaces the old
  `test_name + metric + asset_name` parameters.
- **Error messages** — lowercase, no trailing period, `"could not ..."` phrasing. Shared
  `raise_not_found(entity, name)` helper in `modules/common/errors.py`.

## 3. Migration 004 — Asset Table Fixes + Asset Types

### 3.1 New table: `asset_types`

```
asset_types
  id          UUID     primary key
  name        TEXT     unique  e.g. "vm", "server", "container", "endpoint", "sensor"
  is_default  BOOLEAN  not null, default false
```

Constraints:
- Partial unique index: `CREATE UNIQUE INDEX ON asset_types (is_default) WHERE is_default = true`
  — enforces at most one default at the DB level

Seeded in migration (not from config — simple, works offline):
```
vm       is_default=true
server   is_default=false
container is_default=false
endpoint  is_default=false
sensor    is_default=false
```

### 3.2 Updated `assets` table

- `tags` → `labels` — rename via `alter_column` (preserve data, same pattern as migration 003)
- Add `type_name TEXT NOT NULL DEFAULT 'vm'` — FK → `asset_types.name ON DELETE RESTRICT`
  Default covers existing rows during migration.

### 3.3 Process

1. Update `AssetType` ORM model (new) and `Asset` ORM model (`tags`→`labels`, add `type_name`)
2. `alembic revision --autogenerate -m "004_asset_types_and_fixes"`
3. Correct autogenerated migration: change drop+add for `tags`→`labels` to `alter_column`
4. Apply and verify against live TimescaleDB

> **Deployment note:** The `modules/assets/` router must not be registered in `main.py` until
> migration 004 has been applied. `AssetRead` requires the `labels` and `type_name` columns —
> `from_attributes=True` will raise `AttributeError` against the pre-migration schema.

## 6. Repository Layer

### 6.1 New repositories in `modules/assets/repository.py`

**`AssetTypeRepository`**
- `create(name, is_default) → AssetType`
- `list_all() → list[AssetType]`
- `get_by_name(name) → AssetType | None`
- `set_default(name)` — single transaction: `UPDATE asset_types SET is_default=false`,
  then `UPDATE asset_types SET is_default=true WHERE name=:name`
- `delete(name)` — raises 409 if any asset references `type_name=name`

**`AssetRepository`**
- `create(name, display_name, type_name, labels) → Asset`
- `list_all(type_name?, label_key?, label_val?) → list[Asset]`
- `get_by_name(name) → Asset | None`
- `update(name, **kwargs) → Asset`

**`AssetGroupRepository`**
- `create(name, display_name, description, members, subgroups) → AssetGroupRead`
  — single transaction: insert group, insert members, insert subgroup links
- `get_by_name(name) → AssetGroupRead | None` — eager-loads members + subgroups with asset/group names
- `list_all() → list[AssetGroupRead]`
- `get_tree() → AssetGroupTreeResponse`
  — top_level = groups NOT in `asset_group_links.child_group_id`
- `add_member(group_name, asset_id, weight) → AssetGroupRead`
- `remove_member(group_name, asset_id)`
- `add_subgroup(group_name, child_group_id, weight) → AssetGroupRead`
- `remove_subgroup(group_name, child_group_id)`

**`AssetSLOLinkRepository`** + **`AssetGroupSLOLinkRepository`**
- `create(...)`, `list_by_asset(asset_name)`, `delete(asset_name, link_name)`
- Mirror each other — same pattern, different FK column

### 6.2 Computed fields in `EvaluationSummary` list endpoint

`EvaluationSummary` requires `annotation_count`, `latest_annotation`, and `top_failures` which are
not ORM columns. The evaluation list query must:
- Use a `COUNT` subquery (or window function) for `annotation_count` — never eager-load all annotations
- Use a `LIMIT 1 ORDER BY created_at DESC` correlated subquery for `latest_annotation`
- Derive `top_failures` from `indicator_results` JSONB by filtering array elements where
  `status = 'fail'` in Python after fetch (JSONB array is small per evaluation)

### 6.4 Existing repositories (minor additions)

`SLIRepository`, `SLORepository` — routers added on top, no repo changes. List endpoints return
latest active version per name only (no inactive filtering in Phase 1).

`DataSourceRepository` — two additions required:
- `update(name, display_name?, adapter_url?, labels?) → DataSource` — needed for `PATCH /datasources/{name}`
- `list_all(adapter_type: str | None = None) → list[DataSource]` — add optional filter; existing
  callers pass no argument and behaviour is unchanged

| Method | URL | Request | Response |
|---|---|---|---|
| GET | `/evaluations` | `?asset_name=&slo_name=&result=&date=&group_name=` | `PagedResponse[EvaluationSummary]` |
| GET | `/evaluations/{id}` | — | `EvaluationDetail` |

| PATCH | `/evaluations/{id}/restore` | — | `EvaluationSummary` |
| GET | `/evaluations/{id}/annotations` | — | `list[AnnotationRead]` |
| POST | `/evaluations/{id}/annotations` | `AnnotationCreate` | `AnnotationRead` 201 |
| PATCH | `/evaluations/{id}/annotations/{ann_id}` | `AnnotationUpdate` | `AnnotationRead` |
| DELETE | `/evaluations/{id}/annotations/{ann_id}` | — | 204 |
| GET | `/trend` | `?asset_name=&slo_name=&metric=&limit=50` | `list[TrendPoint]` |

**Trend query logic:**
1. Resolve `asset_id` from `asset_name` via `AssetRepository.get_by_name()`
2. Join: `sli_values.eval_id = evaluations.id`
3. Filter: `evaluations.asset_id = :asset_id AND evaluations.slo_name = :slo_name`
4. Filter: `sli_values.metric_name = :metric`
5. Extract `baseline` by iterating `evaluation.indicator_results` (JSONB array) for the element
   where `element["metric"] == :metric` and reading `element["compared_value"]`.
   `indicator_results` is a list, not a dict — direct key access will not work.
6. Sort: `ORDER BY evaluations.period_start DESC LIMIT :limit` to get the most recent N rows,
   then re-sort the result `ASC` for display (wrap in CTE or subquery).
   A plain `ORDER BY ASC LIMIT N` returns the N oldest, not the N most recent.

**Silent failure mode note:** Never call `EvaluationSummary.model_validate(orm_obj)` with a bare
ORM object. The computed fields `annotation_count`, `latest_annotation`, and `top_failures` have
defaults (`0`, `None`, `[]`) so Pydantic will not raise — it will silently return wrong data.
Always use `model_validate({**orm_obj.__dict__, "annotation_count": n, ...})` with all three
fields explicitly populated.

## 8. Error Handling

```python
# modules/common/errors.py
def raise_not_found(entity: str, name: str) -> NoReturn:
    raise HTTPException(status_code=404, detail=f"{entity} '{name}' not found")

def raise_conflict(detail: str) -> NoReturn:
    raise HTTPException(status_code=409, detail=detail)
```

| Situation | Status |
|---|---|
| Name/ID not found | 404 |
| Duplicate name on create | 409 |
| Delete type/entity in use | 409 |
| Invalid FK reference (type_name unknown) | 422 |
| Soft-delete SLI/SLO | 204 (always allowed) |
