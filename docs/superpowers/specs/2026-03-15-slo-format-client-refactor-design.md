# SLO Format Redesign & Python Client Refactor

**Date:** 2026-03-15
**Status:** Approved

## Summary

Two related improvements driven by the same root cause — the original SLO format was ported
directly from Keptn 1.0 and carries legacy baggage (raw YAML string embedding, `spec_version`,
indicators inside SLOs). This spec covers:

1. **SLO format redesign** — full-stack change from opaque `slo_yaml: str` to structured
   `objectives` + `total_score` + `comparison` at every layer (DB, engine, API, client, manifests)
2. **Python client refactor** — pythonic fixes (imports at top, pydantic models, cleaner error
   handling) and a new `export` CLI command

## Architecture

PostgreSQL is the canonical source of truth. Manifest YAML is an import/export format only —
no files are stored on disk by the service. Users can query and (if needed) directly edit the DB.

## Section 1: Database Schema

### `slo_definitions` (modified)

Drop `slo_yaml TEXT`. Add flat columns for score and comparison:

```
id                      UUID PK
name                    TEXT NOT NULL
version                 INT  NOT NULL          -- auto-incremented per name
active                  BOOL NOT NULL DEFAULT true
display_name            TEXT
notes                   TEXT
author                  TEXT
meta                    JSONB DEFAULT '{}'
total_score_pass_pct    FLOAT NOT NULL DEFAULT 90.0
total_score_warning_pct FLOAT NOT NULL DEFAULT 75.0
comparison              JSONB NOT NULL DEFAULT '{}'  -- rarely extended, sane defaults
created_at              TIMESTAMPTZ NOT NULL DEFAULT now()
```

Unique constraint: `(name, version)`. Indexes: `name`, `(name, version DESC)`.

### `slo_objectives` (new table)

One row per objective per SLO version. Linked by `slo_definition_id` FK.

```
id                UUID PK
slo_definition_id UUID FK → slo_definitions.id  ON DELETE CASCADE
sli               TEXT NOT NULL        -- references SLI definition name at evaluation time
display_name      TEXT NOT NULL DEFAULT ''
weight            INT  NOT NULL DEFAULT 1
key_sli           BOOL NOT NULL DEFAULT false
sort_order        INT  NOT NULL        -- preserves user-defined order
pass_criteria     TEXT[] NOT NULL DEFAULT '{}'   -- e.g. {"<500","<=+10%"}
warning_criteria  TEXT[] NOT NULL DEFAULT '{}'
```

Criteria are flat `TEXT[]` — the Keptn-inherited multi-block AND/OR structure is dropped.
In practice every SLO uses single-block criteria; the complexity added zero real value.

SLO definitions are **immutable** — new objectives are always inserted alongside a new
`slo_definitions` version. Objectives are never updated in place.

### Migration

Autogenerated Alembic migration:
- Add new columns to `slo_definitions`
- Create `slo_objectives` table
- Drop `slo_yaml` column (no existing data to migrate — clean break)

## Section 2: Engine

### `SLO` model (`slo_models.py`)

Drop `spec_version` (redundant with `api_version` in manifests) and `indicators` (indicators
live in the SLI definition, not the SLO):

```python
class SLOObjective(BaseModel):
    sli: str
    display_name: str = ""
    pass_criteria: list[str] = []
    warning_criteria: list[str] = []
    weight: int = 1
    key_sli: bool = False

class SLO(BaseModel):
    objectives: list[SLOObjective]
    comparison: SLOComparison
    total_score: SLOTotalScore
```

### `slo_parser.py`

Repurposed from YAML text parser to a constructor that builds `SLO` from structured data
(DB rows or API payload). No YAML parsing. Validation that objectives reference known
indicators is removed from here — it moves to `AssetSLOLink` creation where both SLI and
SLO are in scope.

### `evaluator.py`

Signature change: `evaluate(slo_yaml: str, ...)` → `evaluate(slo: SLO, ...)`.
No logic changes inside the function — purely mechanical removal of `parse_slo()` call.

## Section 3: API Layer

### Schemas (`slo_registry/schemas.py`)

```python
class SLOObjectiveIn(BaseModel):
    sli: str
    display_name: str = ""
    pass_criteria: list[str] = []
    warning_criteria: list[str] = []
    weight: int = 1
    key_sli: bool = False

class SLODefinitionCreate(BaseModel):
    name: str
    display_name: str | None = None
    objectives: list[SLOObjectiveIn]
    total_score_pass_pct: float = 90.0
    total_score_warning_pct: float = 75.0
    comparison: dict[str, Any] = {}
    notes: str | None = None
    author: str | None = None
    meta: dict[str, Any] = {}

class SLODefinitionRead(BaseModel):
    id: uuid.UUID
    name: str
    display_name: str | None
    version: int
    active: bool
    objectives: list[SLOObjectiveIn]
    total_score_pass_pct: float
    total_score_warning_pct: float
    comparison: dict[str, Any]
    notes: str | None
    author: str | None
    meta: dict[str, Any]
    created_at: datetime
```

`SLOValidateRequest` and `SLOTestRequest` replace `slo_yaml: str` with the same structured
objectives/score fields.

### Repository (`slo_registry/repository.py`)

`create()` inserts into `slo_definitions` then bulk-inserts `slo_objectives` rows.
`get_latest()` and `list_all()` JOIN `slo_objectives` and assemble the full model.

## Section 4: Client + Manifest

### `models.py`

`SLODefinition` drops `slo_yaml`, gains `objectives`, `total_score_pass_pct`,
`total_score_warning_pct`, `comparison`.

### `manifest.py`

All internal types (`ManifestDocument`, `PlanAction`, `ApplyPlan`, `ApplyError`, `ApplyResult`)
converted from `@dataclass` to `pydantic.BaseModel`.

`_create()` / `_has_diff()` / `_update()` handlers for the `SLO` kind updated to pass
`objectives` and `total_score` fields directly.

`_has_diff()` for `SLO`: compare `spec.objectives` and `spec.total_score` against existing
`objectives`, `total_score_pass_pct`, `total_score_warning_pct` from the API.

### `cli.py`

- Inline imports inside `apply()` moved to top of file
- `contextlib.suppress(Exception)` in `_raise_for_status` replaced with explicit try/except
- New `export` command added (see below)

### New manifest format for `SLO` kind

```yaml
api_version: tropek/v1
kind: SLO
metadata:
  name: http-availability-slo
  display_name: HTTP Availability SLO
  author: bootstrap
  notes: Covers P99 latency, error rate, and availability for HTTP services.
spec:
  total_score: {pass_pct: 90.0, warning_pct: 75.0}
  objectives:
    - sli: response_time_p99
      display_name: "Response Time P99"
      pass_criteria: ["<500"]
      warning_criteria: ["<800"]
      weight: 2
    - sli: error_rate
      key_sli: true
      pass_criteria: ["<0.01"]
      warning_criteria: ["<0.05"]
      weight: 3
```

No `slo_yaml`, no `spec_version`, no `indicators`.

### New `export` CLI command

```
tropek export [-f output.yaml] [--base-url URL] [--api-key KEY]
```

Calls existing GET endpoints (assets, asset types, asset groups, data sources, SLI/SLO
definitions, links), reconstructs documents in `_KIND_ORDER` dependency order, writes
multi-document YAML to file or stdout. No new API endpoints required.

### Tests

All inline imports in test files moved to top of file. No logic changes to tests.

## What Does NOT Change

- `SLIDefinition` format — `indicators: dict[str, str]` is already clean
- Evaluation worker flow — only the `evaluate()` call site changes (pass `SLO` model instead
  of constructing one from YAML)
- `AssetSLOLink` / `AssetGroupSLOLink` — unchanged
- Versioning semantics — SLO definitions remain immutable append-only rows
- PyCharm "unresolved reference" — configuration-only fix: set interpreter to workspace
  `.venv` created by `uv sync`. No code changes needed.

---

## Out of Scope

- `comparison` field semantics — defaults are kept, no UI or CLI surface for editing it yet
- SLI cross-validation at link creation time — noted as future work, not blocking this refactor
- Any changes to the evaluation worker or job queue
