## Next Steps — API Layer

The DB layer is the foundation. The next phase is the **API layer**: FastAPI routers and
Pydantic schemas for all new entities, plus refactoring existing evaluation endpoints to
use the new provenance fields and binding-based trigger model.

### Suggested plan order

1. **Pydantic schemas for new entities**
   - `DataSourceCreate` / `DataSourceRead`
   - `SLIDefinitionCreate` / `SLIDefinitionRead` (with `indicators: dict[str, str]`)
   - `AssetGroupCreate` / `AssetGroupRead` (members + group links)
   - `AssetSLOLinkCreate` / `AssetSLOLinkRead`
   - `AssetGroupSLOLinkCreate` / `AssetGroupSLOLinkRead`

2. **New CRUD routers**
   - `POST/GET /datasources` — register and list adapter instances
   - `POST/GET /sli-definitions` — versioned SLI CRUD (mirrors `/slo-definitions`)
   - `POST/GET /asset-groups` — create groups, add members, add child groups
   - `POST/GET /asset-slo-links` and `/asset-group-slo-links` — manage bindings

3. **Refactor evaluation trigger endpoint**
   - Current: caller specifies `slo_name` + `sli_queries` inline
   - New: caller specifies `asset_id` OR `asset_group_id` + optional `binding_name` filter;
     system resolves SLO/SLI/DataSource from binding tables at trigger time
   - Return a `batch_id` when multiple evaluations are spawned

4. **Batch status endpoint**
   - `GET /evaluations/batches/{batch_id}` — aggregate status of all evaluations in a batch

5. **Evaluation response schema updates**
   - Include `period_start`, `period_end` (not `start_time`/`end_time`)
   - Include `sli_name`, `sli_version`, `data_source_name` in response

6. **Worker refactor**
   - Worker currently receives inline SLI queries; refactor to resolve them from `SLIRepository`
     using `sli_name` + `sli_version` stored on the `Evaluation` row

### Key design note for brainstorming

The spec (`docs/superpowers/specs/2026-03-14-tropek-domain-redesign.md`) defines the domain
model in full. The API layer should be driven by it. Before writing a plan, run
`superpowers:brainstorming` to design the API layer and confirm the router/schema structure.
